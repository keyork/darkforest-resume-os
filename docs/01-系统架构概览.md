# 01-系统架构概览

## 1. 项目目标

`darkforest-resume-os` 不是一次性简历生成器，而是一个本地优先的简历工作台：

- 把履历拆成结构化条目
- 把 JD 解析成结构化岗位画像
- 基于事实做匹配分析
- 用多策略生成可继续编辑的 Markdown 简历

## 2. 技术栈

| 层 | 技术 |
| --- | --- |
| Web 框架 | Next.js 14 App Router |
| 语言 | TypeScript |
| UI | Tailwind CSS + shadcn/ui + lucide-react |
| 状态管理 | TanStack Query v5 |
| 参数同步 | nuqs |
| 数据存储 | 浏览器 `localStorage` |
| AI 接入 | 浏览器直连 OpenAI-compatible API |
| 文件解析 | `pdf-parse`、`mammoth` |
| 图表 | Recharts |

## 3. 分层

### 3.1 页面层

位于 `src/app`，负责：

- 页面路由
- 页面级布局
- 任务入口组织

### 3.2 组件层

位于 `src/components`，负责：

- 档案管理 UI
- 匹配分析 UI
- 简历生成 UI
- 全局任务轨道

### 3.3 Hook 层

位于 `src/lib/hooks`，负责：

- 读取和更新浏览器工作区
- 调用 AI agent
- React Query 缓存失效

### 3.4 AI 层

位于 `src/lib/ai`，负责：

- prompt
- context engineering
- zod schema 校验
- 通用 OpenAI-compatible 调用封装

### 3.5 存储与最小服务端层

- `src/lib/client/workspace-storage.ts`：浏览器工作区事实来源
- `src/app/api/profile/import/route.ts`：上传文件后的文本提取

当前服务端不再承担数据库持久化，也不再代理 LLM 请求。

## 4. 关键设计原则

### 4.1 单默认档案模型

当前使用固定档案 ID：

- `profile_default`

这让单用户、本地优先场景更简单直接。

### 4.2 一切履历皆 Item

所有履历内容统一为 `Item`：

- skill
- experience
- project
- education
- certification

统一支持：

- 显隐
- 排序
- 来源标记
- 时间戳

### 4.3 AI 负责推理，工作区负责持久化

典型流程：

1. 前端收集输入
2. 直接调用 AI agent
3. 返回结构化结果
4. 写入浏览器工作区
5. 刷新查询缓存

### 4.4 AI 配置前端持有

配置来源：

- `/settings` 页面
- 浏览器 `localStorage`

这样可以做到：

- 不依赖服务端环境变量
- 不把用户密钥写入后端配置
- 同一部署可连接不同 OpenAI-compatible 服务

## 5. 典型链路

### 5.1 简历导入

1. 用户上传文件或粘贴文本
2. 文件先提取纯文本
3. `profile-agent` 直接由浏览器发起请求
4. 用户预览后导入到工作区

### 5.2 JD 解析

1. 用户输入 JD 文本
2. `jd-parser-agent` 直接解析
3. 结果写入工作区

### 5.3 匹配分析

1. 用户选择一份已解析 JD
2. 前端组合 profile + visible items
3. `match-agent` 输出评分、差距和策略
4. 结果写入工作区

### 5.4 简历生成

1. 用户选择叙事策略、语言和篇幅
2. 可选叠加 JD / Match Result
3. `resume-gen-agent` 生成 Markdown
4. 结果写入工作区
