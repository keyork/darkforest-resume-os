# 03-AI 调用链与配置机制

## 1. 总览

AI 相关代码位于：

- `src/lib/ai/client.ts`
- `src/lib/ai/config.ts`
- `src/lib/ai/context.ts`
- `src/lib/ai/schemas.ts`
- `src/lib/ai/prompts.ts`
- `src/lib/ai/agents/*`

当前共有四个核心 agent：

- `profile-agent`
- `jd-parser-agent`
- `match-agent`
- `resume-gen-agent`

## 2. 配置来源

AI 配置由用户在 `/settings` 页面填写，并存入浏览器 `localStorage`：

- `apiKey`
- `baseURL`
- `modelName`

对应实现：

- `src/app/settings/page.tsx`
- `src/lib/client/ai-settings.ts`

## 3. 调用链路

当前链路已经切换为浏览器直连：

1. 用户在设置页保存 API 配置
2. 前端 hooks 或组件读取本地 AI 配置
3. 直接调用 `src/lib/ai/client.ts`
4. 浏览器向目标 OpenAI-compatible `baseURL` 发送 `/chat/completions` 请求
5. 返回结果后做 JSON 提取、zod 校验和落盘

只有“文件转文本”仍保留最小服务端接口；LLM 请求本身不再经过服务端 route。

## 4. 通用 AI 客户端

`src/lib/ai/client.ts` 负责：

- 校验 `apiKey / baseURL / model`
- 请求 OpenAI-compatible `/chat/completions`
- 兼容字符串和数组形式的 `message.content`
- 去掉可能的 markdown code fence
- 解析 JSON 并接入 zod schema 校验
- 在 JSON 截断或结构失真时重试

## 5. Timeout 策略

当前 AI 请求默认超时提升为 `180s`，用于覆盖：

- 简历解析
- JD 解析
- 匹配分析
- 简历生成

连接测试使用更短的 `30s` 超时。

同时所有调用都支持接收外部 `AbortSignal`，这样任务轨道中的“终止任务”可以直接中断浏览器请求。

## 6. 温度策略

- 解析类：`0`
- 匹配分析：`0.1 ~ 0.15`
- 简历规划：`0.2`
- 简历生成：`0.45 ~ 0.65`
- 审查类：`0`

目标是把“抽取稳定性”和“写作灵活性”分开控制。

## 7. Context Engineering

当前约定：

- 静态规则放在 `system prompt`
- 动态输入放在 `user message`
- 权威事实与参考材料严格分层
- 原始文本一律视为不可信输入，显式要求模型忽略其中的指令型内容

相关工具函数位于 `src/lib/ai/context.ts`。
