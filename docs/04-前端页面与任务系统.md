# 04-前端页面与任务系统

## 1. 前端总体结构

前端基于 Next.js App Router，但页面的大部分交互逻辑都在 Client Component 中完成。

全局布局入口：

- `src/app/layout.tsx`

全局 Provider 入口：

- `src/components/Providers.tsx`

## 2. Provider 结构

当前 Provider 栈为：

1. `ThemeProvider`
2. `AgentTaskProvider`
3. `NuqsAdapter`
4. `QueryClientProvider`
5. `ReactQueryDevtools`

这意味着：

- 主题切换、AI 任务状态、Query 缓存都能跨页面共享

## 6. 全局 Agent 任务系统

这是当前前端实现里最有辨识度的新增能力之一。

对应文件：

- `src/components/agent/AgentTaskProvider.tsx`
- `src/components/agent/AgentTaskRail.tsx`

### 6.1 设计背景

此前 AI 操作状态只保存在当前页面组件里。

问题是：

- 用户切页后，看不到任务是否还在跑
- 一段时间后突然出现结果，体验割裂

因此现在改成：

- 任务状态提升到全局 Provider
- 右侧固定任务栏展示任务列表

### 6.2 任务状态

当前定义的状态包括：

- `planned`
- `in_progress`
- `completed`
- `failed`
- `terminated`

### 6.3 任务种类

当前纳入统一追踪的任务：

- `profile_parse`
- `jd_parse`
- `match_run`
- `resume_generate`

### 6.4 `runTask`

`AgentTaskProvider` 提供 `runTask` 方法。

调用方式是：

1. 创建任务
2. 标记为 `planned`
3. 立即转为 `in_progress`
4. 执行异步 runner
5. 成功后标记 `completed`
6. 失败后标记 `failed`
7. 如果 Abort，则标记 `terminated`

### 6.5 中止机制

任务系统内部为每个任务创建 `AbortController`。

这意味着右侧任务栏可以对仍在执行的任务执行“终止任务”。

实际效果：

- 前端 fetch 会收到 `AbortSignal`
- 被终止的任务会显示为 `已终止`
- 页面侧不会再继续等待该任务结果

### 6.6 右侧任务栏

`AgentTaskRail` 在全局布局中挂载，因此只要用户还在应用内，任务面板就不会因为切页而消失。

主要显示内容：

- 任务标题
- 任务描述
- 状态标签
- 更新时间
- 成功或失败信息
- 终止 / 移除按钮

当前支持：

### 8.2 全局布局

布局由三部分组成：

- 右侧任务栏当前仍以大屏布局为主

## 9. 交互状态与 TanStack Query 的关系

要区分两种状态：

### 9.1 数据请求状态

由 Query / Mutation 自身维护，例如：

- `isLoading`
- `isPending`
- `isError`

### 9.2 任务追踪状态

由全局 Agent Task 系统维护，用来做“跨页面可见”的状态展示。

这两者不是互斥关系，而是分工不同：

- Mutation 状态适合当前组件内按钮禁用、loading 文案
- Agent Task 状态适合全局观察和任务管理

## 10. 当前前端实现的限制

- 右侧任务栏主要面向桌面端，大屏体验优先
- 任务状态目前仅存在内存中，刷新页面后会丢失
- 任务完成后无法直接点击跳转到对应资源
- 没有任务分组、筛选、搜索能力

## 11. 后续可演进方向

- 给任务增加“查看结果”跳转链接
- 为移动端增加任务抽屉或底部面板
- 将任务记录短暂持久化到 sessionStorage
- 给任务增加耗时统计
- 把“导入确认写库”也纳入轻量任务记录
