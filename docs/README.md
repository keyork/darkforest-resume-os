## 当前实现的关键结论

- 项目当前是单用户、本地优先形态，围绕默认档案 `profile_default` 建模
- AI 设置通过前端页面保存到浏览器，再随请求头发送给服务端
- JD 解析、匹配分析、简历生成、简历导入都接入了统一任务系统
- 当前前端主流程以浏览器工作区存储为主；仓库中仍保留 SQLite / Drizzle 数据层与相关 API
- 技术文档更偏“代码目前是怎么工作的”；产品文档更偏“这个产品为什么这样设计、接下来该往哪走”
