# 08 · 功能设计：数据管理

本文档覆盖四个数据层面的功能：

1. 数据导出与备份
2. PDF 直接导出
3. 全局搜索
4. 对话式档案录入

---

## 一、数据导出与备份

### 背景与风险

当前所有档案数据都保存在浏览器 `localStorage`。对本地优先产品来说这很轻量，但也有明显限制：

- 清理浏览器站点数据会丢失工作区
- 更换浏览器或设备时不会自动同步
- 无痕模式下数据不可持续保留

### 导出功能

入口建议放在设置页，提供“导出当前工作区”按钮。

导出结构直接对应 `WorkspaceSnapshot`：

```json
{
  "version": 1,
  "profile": {},
  "items": [],
  "jds": [],
  "matchResults": [],
  "generatedResumes": []
}
```

推荐实现：

- 前端直接读取 `workspace-storage.ts`
- 生成 JSON Blob
- 用 `URL.createObjectURL` 触发下载

### 导入功能

入口与导出放在同一卡片中，支持：

- 覆盖当前工作区
- 合并导入

导入前需要做：

- JSON 结构校验
- 版本检查
- 摘要预览
- 覆盖模式二次确认

---

## 二、PDF 直接导出

当前“简历生成”页输出 Markdown，用户仍需要自行转成 PDF。

短期推荐：

- 使用浏览器打印样式
- 提供“下载 PDF”按钮
- 调用 `window.print()`

中期可考虑：

- `@react-pdf/renderer`

---

## 三、全局搜索

随着工作区数据增加，需要支持快速搜索：

- skills
- experiences
- projects
- educations
- certifications
- JDs
- generated resumes

推荐方案：

- 前端本地索引
- `Ctrl/Cmd + K` 触发
- 不依赖服务端

---

## 四、对话式档案录入

在当前手动录入和整份简历导入之间，增加一条更轻量的入口：

- 用户输入自然语言经历
- 前端直接调用 AI 解析
- 返回结构化 Item 预览
- 用户确认后落盘到工作区

这条能力与当前浏览器直连 LLM 的架构是兼容的。
