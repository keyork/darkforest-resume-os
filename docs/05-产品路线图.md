# 05 · 产品路线图与功能规划

本文档是基于现有四条核心链路（档案管理 → JD 匹配 → 简历生成 → 设置）的延展规划，记录尚未实现但值得做的功能方向。

---

## 当前已实现功能一览

| 模块 | 能力 |
|---|---|
| 档案管理 | 结构化录入技能/经历/项目/教育/证书，支持 AI 解析导入，拖拽排序，可见性控制 |
| JD 匹配分析 | AI 解析 JD、五维评分、需求匹配表、差距分析、简历策略建议 |
| 简历生成 | 多叙事策略、中英文、1/2 页，关联 JD 和匹配结果，Markdown 输出 |
| 设置 | 浏览器侧 AI 配置（API Key / Base URL / 模型名），连接测试 |

---

## 规划功能清单

### P0 — 修复现有短板（影响基础可用性）

#### P0-1 数据导出与备份
**现状问题：** 所有档案数据当前保存在浏览器 `localStorage`。用户清理站点数据、切换浏览器或更换设备后，工作区不会自动保留。
**目标：** 支持一键导出全量数据为 JSON，支持从 JSON 导入还原。
→ 详见 [08-功能设计-数据管理](./08-功能设计-数据管理.md)

#### P0-2 投递追踪看板
**现状问题：** 用户已经在系统里有了 JD 和简历，但"我把哪份简历发给了哪家公司、现在进展到哪一步"完全在系统之外管理（用电子表格或者脑子记）。这是求职流程中最关键的一环，数据已经齐备，只差一个 UI。
→ 详见 [06-功能设计-投递追踪](./06-功能设计-投递追踪.md)

---

### P1 — 高频需求，技术难度适中

#### P1-1 档案条目 AI 润色
在每条档案条目（经历、项目等）上加「润色」入口，AI 按 STAR/量化原则改写描述，用户确认后替换原文。比整份简历重新导入颗粒度更细，用户控制感更强。
→ 详见 [07-功能设计-AI增强](./07-功能设计-AI增强.md)

#### P1-2 简历健康度评分
不依赖 JD，直接对当前档案做整体质量评估：完整度、量化表达比例、关键词丰富度、时间线连贯性。适合用户在"还没想好投哪里"时用来发现档案短板。
→ 详见 [07-功能设计-AI增强](./07-功能设计-AI增强.md)

#### P1-3 批量 JD 对比分析
一次粘贴 3-5 个 JD，并行分析后出横向对比表：匹配度排名、各岗位核心差距、投递优先级建议。解决用户"不知道先投哪个"的决策问题。
→ 详见 [07-功能设计-AI增强](./07-功能设计-AI增强.md)

---

### P2 — 完成最后一公里

#### P2-1 PDF 直接导出
当前生成的 Markdown 需要用户自己转 PDF，门槛较高。提供浏览器打印样式或集成轻量渲染方案，让"复制简历"变成"下载 PDF"。
→ 详见 [08-功能设计-数据管理](./08-功能设计-数据管理.md)

#### P2-2 面试问题预测
匹配分析完成后，基于差距项和高风险需求，生成一份"高概率被问到的面试题"及参考回答思路，帮助用户把匹配分析转化为面试准备。
→ 详见 [07-功能设计-AI增强](./07-功能设计-AI增强.md)

#### P2-3 全局搜索
`⌘K` 快速搜索所有档案条目、历史 JD、生成简历，本地 fuse.js 模糊搜索，无需服务端。
→ 详见 [08-功能设计-数据管理](./08-功能设计-数据管理.md)

---

### P3 — 体验升级，较高工程量

#### P3-1 对话式档案录入
用自然语言告诉系统"我在 XX 公司做了 XX 项目，负责 XX，结果是 XX"，AI 自动解析成结构化条目，用户一键确认。比填表单快，适合懒得手动录入的用户。
→ 详见 [08-功能设计-数据管理](./08-功能设计-数据管理.md)

#### P3-2 目标岗位市场分析
聚合用户关注的多个 JD 的关键词，生成"当前市场对你感兴趣方向的核心技能需求图谱"，并和个人档案做交叉，告诉用户哪些技能要补、哪些已超出平均水位。

#### P3-3 差距学习路径
在差距分析里，对每个高优先级 gap，AI 给出具体的补足建议：推荐学什么、做什么项目、大概需要多久。把"我和岗位有差距"转化为"我知道怎么补"。

## 优先级矩阵

| 功能 | 用户价值 | 实现成本 | 优先级 |
|---|---|---|---|
| 数据导出备份 | 高（防止数据丢失） | 低 | **P0** |
| 投递追踪看板 | 高（补完求职闭环） | 中 | **P0** |
| 档案条目 AI 润色 | 高（高频使用） | 低 | **P1** |
| 简历健康度评分 | 高（无 JD 时的核心价值） | 中 | **P1** |
| 批量 JD 对比 | 中高（决策辅助） | 中 | **P1** |
| PDF 直接导出 | 高（完成最后一公里） | 中 | **P2** |
| 面试问题预测 | 中高（延伸价值链） | 低 | **P2** |
| 全局搜索 | 中（效率提升） | 低 | **P2** |
| 对话式录入 | 中（体验升级） | 高 | **P3** |
| 市场分析 | 中（战略视角） | 高 | **P3** |
| 差距学习路径 | 中（增强差距分析） | 中 | **P3** |

## 技术债与已知问题

在推进新功能前，以下技术问题值得先处理：

1. **工作区备份缺失** — 当前依赖浏览器本地存储，建议先做导出 / 导入功能
2. **AI 配置在 localStorage** — 用户换浏览器/无痕窗口时需重新配置，考虑提供配置文件导入/导出
3. **无用户认证** — 当前是单用户系统（`profile_default`），如果将来多用户化需要较大改造
4. **Markdown 渲染依赖外部** — 生成的简历需要用户自己找工具渲染，PDF 导出是刚需
